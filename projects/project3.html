Autonomous Indoor Navigation Robot

SIANV is an indoor autonomous robot developed as a final-year semester project (2024–25). It is designed to navigate autonomously in indoor environments using path planning and obstacle avoidance.SIANV relies on a 2D LiDAR for LaserScan and for odometry estimation scan matching techniques. We used Google’s Cartographer SLAM for mapping and localization in our test area. Although we initially implemented AMCL (Adaptive Monte Carlo Localization) for position tracking, we later removed it since the robot did not use multiple sensors. With a high-frequency LiDAR, we were able to estimate the robot’s pose and obtain reliable data without needing additional sensors. The broader idea was to transform an industrial electric vehicle into an autonomous robot with minimal physical modifications—primarily through computational techniques and the use of necessary sensors.

We used ROS 2 with the Nav2 navigation stack, which provided a practical and efficient framework. It used a hybrid A* global planner for route planning and a local planner for real-time obstacle avoidance. On the software side, we developed several computation nodes to handle sensor processing, motor control, system state publishing, and communication between hardware and software. We also implemented a custom reactive planner for unstructured environments without prebuilt maps which i talked about in the next paragraph, as well as a voice-assisted navigation mode. I tested SIANV in a lab and at home, where it navigated autonomously and avoided obstacles smoothly. The entire project was built in a duration of a month and two. However, there are a lot of areas of improvement.

As of now, I would certainly work with my other project and give it a cool end. I might make changes to it in the future. It was incredibly rewarding to see a robot I built operate smoothly: this final-year project taught me a ton about SLAM, localization, path planning, and real-world system integration.

Reactive Planner (the non pre-built map technique )

The reactive planner approach used proportional control for both direction and distance to reach the target. The robot avoids obstacles only if they lie within its trajectory, using a field of view (FOV) approach. The FOV varies depending on the size of the robot, typically around 120 degrees or 90 degrees. This FOV is divided into two halves: the left FOV and the right FOV. If an obstacle is detected in one half, the robot will turn through the other half to avoid it, and vice versa. If obstacles are present in both halves, the planner identifies the boundaries as bypass nodes. The planner then compares the distances of the possible bypass nodes and selects the one that brings the robot closer to the target, navigating through that half. Once the direction is changed, the robot continues to use the left or right half approach to avoid the obstacle.

Voice-Controlled Mode

The voice-controlled mode enables intuitive human-robot interaction by allowing the user to issue verbal commands to guide the robot’s motion. Instead of relying on manual control through a keyboard or joystick, the robot interprets spoken instructions and converts them into navigation commands.

This feature leverages speech recognition to detect predefined keywords such as “forward”, “backward”, “left”, “right”, “stop” and complex commands. Upon recognizing a command, the system publishes the appropriate velocity message to the /cmd_vel topic, enabling the robot to execute the desired movement.
